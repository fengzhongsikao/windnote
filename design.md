一、 整体设计风格与基调
基于 HeroUI 的特性，软件为“极简玄学”风格。
整体色彩模仿项目下的 image.png

    色彩系统 (Color)：
        主色调：建议使用 深空黑 (Dark Mode 默认) 或 宣纸白。
        强调色：利用 HeroUI 的色彩体系，使用 赤红 (Red) 代表阳/动，玄黑 (Blue/Black) 代表阴/静。也可以使用 金色 (Yellow/Angel) 代表吉/结果。
        背景：使用磨砂玻璃（Blur）效果，叠加若隐若现的八卦或云纹背景图。
    字体 (Typography)：
        标题使用具有书法感的字体（如“演示悠然小楷”），正文使用清晰的无衬线字体（如 Inter, system-ui）。
    动效 (Motion)：
        页面切换使用平滑的淡入淡出或滑动。
        算卦结果生成时，使用“打字机”效果逐字显示，配合轻微的震动反馈。

二、 信息架构与页面跳转逻辑
为了保证桌面端的沉浸感，建议采用侧边栏导航 (Sidebar) + 主内容区的布局，这是 HeroUI 非常擅长的模式。
1. 一级页面（主导航）

    首页 (Dashboard)：显示今日运势摘要、最近的占卜记录、启动按钮。
    六爻起卦 (I Ching / Liu Yao)：核心功能页。
    梅花易上下文 (Meihua Yishu)：核心功能页。
    解卦库 (Library)：查阅卦象解释、历史案例。
    设置 (Settings)：AI 模型选择、通知设置。

2. 二级页面（功能流程）

    起卦方式选择：数字起卦、时间起卦、随机起卦。
    排盘详情：展示生成的卦象（上卦、下卦、变卦）。
    爻位编辑：针对六爻，允许手动调整动爻。

3. 三级页面（深度交互）

    爻位详情：点击具体的爻，查看该爻的详细解释。
    历史记录详情：查看某一次具体的占卜对话。

三、 具体页面设计与交互流程
1. 首页 (Dashboard)

    布局：HeroUI 的 Cards 网格布局。
    内容：
        大卡片：今日黄历、财神方位（利用你所在的城市“广东中山”获取地理位置信息）。
        快捷按钮：悬浮的“立即起卦”按钮 (FAB)。
        最近记录：横向滚动的卡片列表，点击跳转到详情。

2. 六爻 (Liu Yao) 功能页

    交互流程：
        输入：用户输入所问之事（Text Input）。
        起卦：点击按钮，触发 Loading 状态（HeroUI 的 Spinner）。
        结果展示：
            使用 Table 或 Custom Grid 展示六爻排盘。
            每一爻是一个可点击的 Chip 或 Card。
            动爻用 红色 高亮显示。
    AI 解析：下方是一个 Chat UI，显示 AI 根据卦象生成的分析（利用 HeroUI 的 Chat 组件）。

3. 梅花易数 (Meihua Yishu) 功能页

    特色：梅花重在“外应”，即随机性。
    设计：
        万物类象：提供一个图片墙，用户看到哪张图就点哪张（如看到猫、车、树）。
        数字/文字：输入汉字，系统计算笔画数起卦。
        结果：展示体卦、用卦、互卦的关系图（可使用 HeroUI 的 Diagram 或简单的 Div 布局）。

四、 弹窗 (Modal) 与反馈机制
HeroUI 的 Modal 组件非常强大，适合用于关键操作。

    起卦确认弹窗：
        触发：点击“开始摇卦”。
        内容：确认所问之事是否准确？“心诚则灵，确定以此问题起卦吗？”
        按钮：取消 / 确定摇卦。
    结果详解弹窗：
        触发：点击排盘中的某一爻。
        内容：该爻的原文、白话解释、AI 的个性化建议。
        风格：使用 size="5xl" 的大弹窗，内部包含 Tabs（原文 / 解释 / 案例）。
    通知 (Toasts)：
        场景：复制卦象成功、保存记录成功。
        样式：使用 HeroUI 的 Toast 组件，位置在右上角。

五、 技术实现建议 (基于 HeroUI)
由于你指定了 HeroUI，以下是一些技术层面的建议，帮助你快速落地：

    组件选择：
        布局：使用 Layout 组件构建侧边栏和头部。
        卡片：使用 Card 组件展示卦象，通过 className 添加自定义的旋转和阴影效果。
        表格：使用 Table 组件展示六爻的列表，使用 TableRow 的 isSelected 属性高互动爻。
        主题：利用 useTheme 钩子，允许用户在“玄黑”和“素白”之间切换。
    暗黑模式 (Dark Mode)：
        算卦软件在暗光环境下使用频率较高，务必开启 HeroUI 的 dark 模式支持，减少用户眼睛疲劳。
    响应式：
        虽然是桌面端，也要考虑不同分辨率的适配，HeroUI 基于 Tailwind CSS，可以轻松使用 md:, lg: 前缀调整布局。

六、 截图保存功能设计
针对你提到的“算卦的截图可以保存在本地”的需求，可以设计如下交互：

    触发按钮：在结果页面的右上角放置一个 HeroUI 风格的 Button，带有相机图标，文字为“保存结果”。
    前端逻辑：
        使用 html2canvas 或 dom-to-image 库。
        将包含卦象和AI解析的 Card 或 Div 区域渲染为 Canvas 或图片数据。
    保存逻辑：
        利用或浏览器的 download 属性。
        文件名建议格式：卦名_日期.png (例如：乾为天_20260512.png)。
    反馈：点击后弹出 Toast 提示“截图已保存至本地”。

七.技术 桌面端 使用了wails v2，包管理器使用 deno，按照需要使用 react流行的模块比如 react-router.
